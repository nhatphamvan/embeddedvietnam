Clock Tree là gì?

Clock Tree là hệ thống phân phối xung nhịp (clock) từ nguồn oscillator đến các ngoại vi và CPU trong vi điều khiển. Mỗi nhánh của cây clock có thể được cấu hình độc lập về tần số và trạng thái (on/off).

💡 Tại sao cần Clock Tree? Vì mỗi peripheral (UART, SPI, ADC…) có yêu cầu tần số hoạt động khác nhau. Clock Tree cho phép CPU chạy 180 MHz trong khi I2C chỉ cần 8 MHz.

Các thành phần chính

  • HSI (High Speed Internal) — dao động RC nội, ~16 MHz, không cần thạch anh ngoài
  • HSE (High Speed External) — thạch anh ngoài, chính xác hơn, thường 8–25 MHz
  • LSI / LSE — nguồn clock chậm cho RTC và WDT (~32 kHz)
  • PLL (Phase-Locked Loop) — nhân tần số từ HSI/HSE lên tần số cao (vd: 180 MHz)
  • Prescaler — chia tần số cho từng bus (AHB, APB1, APB2)

Bus clock trong STM32

Sau khi qua PLL, clock được phân phối qua các bus với bộ chia (prescaler):

  • SYSCLK — clock hệ thống (max 180 MHz trên F4)
  • AHB — Advanced High-Performance Bus (CPU, DMA, memory)
  • APB1 — tần số thấp hơn, dùng cho UART, SPI, I2C, TIM2-7
  • APB2 — tần số cao hơn APB1, dùng cho ADC, TIM1, SPI1
⚠️ Lỗi phổ biến: Quên enable clock cho peripheral trước khi sử dụng. Trong AUTOSAR MCAL, Mcu_Init() phải được gọi đầu tiên trước tất cả driver khác.

Trong AUTOSAR MCAL

Module Mcu (AUTOSAR SWS_Mcu) chịu trách nhiệm cấu hình clock tree thông qua cấu trúc Mcu_ClockSettingConfigType. Các setting được sinh ra bởi tool cấu hình (vd: EB Tresos, MCAL Configurator).

/* Mcu_Cfg.h — generated by MCAL Configurator */
#ifndef MCU_CFG_H
#define MCU_CFG_H

/* Clock setting index */
#define MCU_CLOCK_SETTING_PLL_180MHZ   ((Mcu_ClockType)0U)
#define MCU_CLOCK_SETTING_HSI_16MHZ    ((Mcu_ClockType)1U)

/* RAM section index */
#define McuConf_McuRamSectorSettingConf_McuRamSector0  ((Mcu_RamSectionType)0U)

#endif /* MCU_CFG_H */

/* Mcu.c — Clock tree configuration for STM32F4 */

static const Mcu_ClockSettingConfigType Mcu_ClockSettingConfig[] = {
    /* Index 0: PLL → 180 MHz */
    {
        .McuClockSettingId     = MCU_CLOCK_SETTING_PLL_180MHZ,
        .McuClockSourceSelect  = MCU_CLOCK_SOURCE_HSE,   /* HSE 8 MHz */
        .McuPllM               = 4U,    /* VCO input = 8/4 = 2 MHz */
        .McuPllN               = 180U,  /* VCO output = 2*180 = 360 MHz */
        .McuPllP               = 2U,    /* SYSCLK = 360/2 = 180 MHz */
        .McuPllQ               = 8U,    /* USB = 360/8 = 45 MHz */
        .McuAHBPrescaler       = MCU_AHB_DIV_1,   /* AHB  = 180 MHz */
        .McuAPB1Prescaler      = MCU_APB1_DIV_4,  /* APB1 = 45  MHz */
        .McuAPB2Prescaler      = MCU_APB2_DIV_2,  /* APB2 = 90  MHz */
        .McuFlashWaitStates    = 5U,               /* required at 180 MHz */
    },
};

void Mcu_Init(const Mcu_ConfigType* ConfigPtr)
{
    /* 1. Enable HSE and wait ready */
    RCC->CR |= RCC_CR_HSEON;
    while (!(RCC->CR & RCC_CR_HSERDY));

    /* 2. Configure PLL */
    RCC->PLLCFGR = (ConfigPtr->ClockSetting->McuPllM)
                 | (ConfigPtr->ClockSetting->McuPllN  << 6U)
                 | (((ConfigPtr->ClockSetting->McuPllP / 2U) - 1U) << 16U)
                 | RCC_PLLCFGR_PLLSRC_HSE
                 | (ConfigPtr->ClockSetting->McuPllQ  << 24U);

    /* 3. Enable PLL */
    RCC->CR |= RCC_CR_PLLON;
    while (!(RCC->CR & RCC_CR_PLLRDY));

    /* 4. Switch SYSCLK to PLL */
    RCC->CFGR = (RCC->CFGR & ~RCC_CFGR_SW) | RCC_CFGR_SW_PLL;
    while ((RCC->CFGR & RCC_CFGR_SWS) != RCC_CFGR_SWS_PLL);
}

/* main.c — AUTOSAR BSW initialization sequence */
#include "Mcu.h"
#include "Port.h"

/* Generated config structs */
extern const Mcu_ConfigType   McuConfigSet;
extern const Port_ConfigType  PortConfigSet;

int main(void)
{
    /* Step 1: Initialize MCU (clock tree, RAM, reset reason) */
    Mcu_Init(&McuConfigSet);

    /* Step 2: Activate clock setting (PLL 180 MHz) */
    Mcu_InitClock(MCU_CLOCK_SETTING_PLL_180MHZ);

    /* Step 3: Wait until PLL is stable */
    while (Mcu_GetPllStatus() != MCU_PLL_LOCKED);

    /* Step 4: Distribute clock to all peripherals */
    Mcu_DistributePllClock();

    /* Step 5: Initialize Port driver (GPIO direction, mode) */
    Port_Init(&PortConfigSet);

    /* ... initialize other drivers ... */

    for (;;) {
        /* main loop */
    }
}
❓ Kiểm tra kiến thức ▾
1 PLL trong vi điều khiển có chức năng gì?
✓ Đúng! PLL (Phase-Locked Loop) nhân tần số đầu vào — ví dụ 8 MHz × 22.5 = 180 MHz.
✗ Sai. PLL có nhiệm vụ NHÂN tần số (multiply), không phải chia. Bộ chia là Prescaler.
2 Trong AUTOSAR, hàm nào phải được gọi đầu tiên để khởi tạo clock tree?
✓ Đúng! Mcu_Init() luôn là hàm đầu tiên trong chuỗi khởi tạo BSW.
✗ Sai. Theo AUTOSAR SWS, Mcu_Init() phải được gọi trước tất cả các driver khác.
3 APB1 và APB2 khác nhau thế nào trong STM32F4 @ 180 MHz?
✓ Đúng! APB1 ÷4 = 45 MHz, APB2 ÷2 = 90 MHz. APB2 chạy nhanh hơn, dùng cho ADC, SPI1.
✗ Sai. APB1 (÷4=45MHz) CHẬM hơn APB2 (÷2=90MHz).
