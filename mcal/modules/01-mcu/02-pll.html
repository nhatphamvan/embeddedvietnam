PLL — Phase Locked Loop

Hiểu cơ chế PLL nhân tần số từ nguồn clock thô (HSE/HSI) lên tần số hệ thống cao, cách tính toán các hệ số M/N/P, và triển khai theo chuẩn AUTOSAR MCAL.

📖 Lý thuyết PLL là gì và tại sao cần nó? ›

1. Vấn đề: tần số crystal quá thấp

Vi điều khiển như STM32F4 có thể chạy tới 168 MHz, nhưng crystal HSE thường chỉ có tần số 8–25 MHz. Nếu dùng trực tiếp, CPU sẽ chậm hơn 8–20× so với khả năng tối đa.

PLL (Phase Locked Loop) là mạch điện tử bên trong chip cho phép nhân tần số từ nguồn tham chiếu thấp lên tần số mục tiêu cao, đồng thời giữ ổn định pha.

2. Nguyên lý hoạt động của PLL

PLL gồm 4 khối chính:

  • Phase Detector (PD) — so sánh pha giữa tín hiệu tham chiếu và tín hiệu phản hồi
  • Low-Pass Filter (LPF) — lọc nhiễu tần số cao từ đầu ra PD
  • VCO (Voltage Controlled Oscillator) — tạo dao động tần số cao, điều chỉnh bởi điện áp từ LPF
  • Divider (N) — chia tần số VCO xuống, đưa về PD để so sánh (vòng phản hồi)

Khi vòng khoá (lock), tần số phản hồi bằng tần số tham chiếu, do đó: f_VCO = f_ref × N

3. Công thức PLL trên STM32F4

STM32F4 sử dụng công thức PLL chính:

f_PLL = (f_in / PLLM) × PLLN / PLLP với f_in = HSE hoặc HSI (÷2)
Tham sốPhạm viVai trò
PLLM2–63Pre-divider — chia f_in xuống ~1–2 MHz (đầu vào VCO)
PLLN50–432VCO multiplier — nhân lên, phải nằm trong 100–432 MHz
PLLP2, 4, 6, 8Post-divider — chia VCO xuống SYSCLK ≤ 168 MHz
PLLQ2–15USB/SDIO clock — phải = 48 MHz cho USB hoạt động

4. Ví dụ tính toán — 168 MHz từ HSE 8 MHz

1
Chọn PLLM = 8

f_vco_in = 8 MHz / 8 = 1 MHz (tối ưu cho VCO)

2
Chọn PLLN = 336

f_vco = 1 MHz × 336 = 336 MHz (trong giới hạn 100–432 MHz)

3
Chọn PLLP = 2

f_SYSCLK = 336 MHz / 2 = 168 MHz ✓

4
Kiểm tra PLLQ cho USB

PLLQ = 7 → 336 / 7 = 48 MHz ✓

Lưu ý AUTOSAR: Tất cả giá trị M, N, P, Q phải được khai báo trong Mcu_Cfg.h dưới dạng macro, không được hardcode trong source code. Đây là yêu cầu của tiêu chuẩn về tính cấu hình (configurability).

5. Flash Latency — bắt buộc phải set trước khi tăng clock

Flash memory trên STM32 có latency cố định (~30ns). Khi tăng SYSCLK, CPU đọc flash nhanh hơn nhưng flash không kịp trả dữ liệu → đọc sai. Phải cấu hình wait states trước:

SYSCLK (VDD = 3.3V)Flash Latency (WS)
0 – 30 MHz0 WS
30 – 60 MHz1 WS
60 – 90 MHz2 WS
90 – 120 MHz3 WS
120 – 150 MHz4 WS
150 – 168 MHz5 WS

💻 Code Cấu hình PLL theo chuẩn AUTOSAR MCAL › 
/**
 * Mcu_Cfg.h — PLL Configuration (AUTOSAR 4.x)
 * Target: STM32F407 @ 168 MHz from HSE 8 MHz
 */
#ifndef MCU_CFG_H
#define MCU_CFG_H

/* ── PLL Source ──────────────────────────────── */
#define MCU_PLL_SRC         RCC_PLLSOURCE_HSE   /* HSE = 8 MHz */

/* ── PLL Multiplier/Dividers ─────────────────── */
#define MCU_PLLM            8U    /* f_vco_in = 8/8  = 1 MHz   */
#define MCU_PLLN            336U  /* f_vco    = 1*336 = 336 MHz  */
#define MCU_PLLP            2U    /* SYSCLK   = 336/2 = 168 MHz  */
#define MCU_PLLQ            7U    /* USB CLK  = 336/7 = 48 MHz   */

/* ── Bus prescalers ──────────────────────────── */
#define MCU_AHB_PRESCALER   RCC_SYSCLK_DIV1    /* HCLK  = 168 MHz */
#define MCU_APB1_PRESCALER  RCC_HCLK_DIV4      /* PCLK1 =  42 MHz (max 42) */
#define MCU_APB2_PRESCALER  RCC_HCLK_DIV2      /* PCLK2 =  84 MHz (max 84) */

/* ── Flash wait states ───────────────────────── */
#define MCU_FLASH_LATENCY   FLASH_LATENCY_5     /* 5 WS for 168 MHz, 3.3V */

/* ── MCU Clock modes ─────────────────────────── */
typedef enum {
    MCU_CLOCK_MODE_HSI     = 0x00U,
    MCU_CLOCK_MODE_HSE     = 0x01U,
    MCU_CLOCK_MODE_PLL     = 0x02U,
} Mcu_ClockModeType;

#define MCU_CLOCK_MODE_DEFAULT   MCU_CLOCK_MODE_PLL

#endif /* MCU_CFG_H */

/**
 * Mcu.c — Mcu_InitClock() implementation
 * Theo AUTOSAR SWS_Mcu_00210: hàm phải set flash latency
 * TRƯỚC khi switch sang PLL, để tránh đọc flash sai.
 */
#include "Mcu.h"
#include "Mcu_Cfg.h"

Std_ReturnType Mcu_InitClock(Mcu_ClockType ClockSetting)
{
    RCC_OscInitTypeDef osc = {0};
    RCC_ClkInitTypeDef clk = {0};

    /* Step 1 — Bật HSE và cấu hình PLL */
    osc.OscillatorType = RCC_OSCILLATORTYPE_HSE;
    osc.HSEState       = RCC_HSE_ON;
    osc.PLL.PLLState   = RCC_PLL_ON;
    osc.PLL.PLLSource  = MCU_PLL_SRC;
    osc.PLL.PLLM       = MCU_PLLM;   /* 8  */
    osc.PLL.PLLN       = MCU_PLLN;   /* 336 */
    osc.PLL.PLLP       = MCU_PLLP;   /* 2  */
    osc.PLL.PLLQ       = MCU_PLLQ;   /* 7  */

    if (HAL_RCC_OscConfig(&osc) != HAL_OK) {
        return E_NOT_OK;
    }

    /* Step 2 — Flash latency PHẢI set trước khi tăng SYSCLK */
    __HAL_FLASH_SET_LATENCY(MCU_FLASH_LATENCY);
    /* Đọc lại để xác nhận đã ghi thành công (ARM write buffer) */
    if (__HAL_FLASH_GET_LATENCY() != MCU_FLASH_LATENCY) {
        return E_NOT_OK;
    }

    /* Step 3 — Switch bus clocks sang PLL */
    clk.ClockType      = (RCC_CLOCKTYPE_SYSCLK | RCC_CLOCKTYPE_HCLK |
                           RCC_CLOCKTYPE_PCLK1  | RCC_CLOCKTYPE_PCLK2);
    clk.SYSCLKSource   = RCC_SYSCLKSOURCE_PLLCLK;
    clk.AHBCLKDivider  = MCU_AHB_PRESCALER;   /* ÷1  → 168 MHz */
    clk.APB1CLKDivider = MCU_APB1_PRESCALER;  /* ÷4  →  42 MHz */
    clk.APB2CLKDivider = MCU_APB2_PRESCALER;  /* ÷2  →  84 MHz */

    if (HAL_RCC_ClockConfig(&clk, MCU_FLASH_LATENCY) != HAL_OK) {
        return E_NOT_OK;
    }

    return E_OK;
}

/* Mcu_DistributePllClock() — unlock PLL output tới peripherals */

    /* Trên STM32 HAL, clock đã được phân phối sau HAL_RCC_ClockConfig.

/**
 * Mcu_FlashLatency.c — Tại sao phải set TRƯỚC khi tăng clock?
 *
 * Thứ tự ĐÚNG:
 *   1. Set Flash Latency = 5 WS
 *   2. Tăng SYSCLK lên 168 MHz (switch sang PLL)
 *
 * Thứ tự SAI (dễ gây bug khó tìm):
 *   1. Tăng SYSCLK lên 168 MHz
 *   2. Set Flash Latency
 *   → Trong khoảng thời gian giữa 2 bước, CPU đọc flash
 *     với 168 MHz nhưng chỉ 0 WS → đọc INSTRUCTION SAI
 *     → undefined behavior, hard fault, hoặc chạy code lạ.
 */

/* Macro kiểm tra flash latency đúng theo SYSCLK (3.3V) */
#define MCU_GET_FLASH_LATENCY(sysclk_mhz) \
    ((sysclk_mhz) <=  30 ? FLASH_LATENCY_0 : \
     (sysclk_mhz) <=  60 ? FLASH_LATENCY_1 : \
     (sysclk_mhz) <=  90 ? FLASH_LATENCY_2 : \
     (sysclk_mhz) <= 120 ? FLASH_LATENCY_3 : \
     (sysclk_mhz) <= 150 ? FLASH_LATENCY_4 : \
                            FLASH_LATENCY_5)

/* Áp dụng: */
static Std_ReturnType Mcu_SetFlashLatency(uint32 SysClkMhz)
{
    uint32 ws = MCU_GET_FLASH_LATENCY(SysClkMhz);
    __HAL_FLASH_SET_LATENCY(ws);
    if (__HAL_FLASH_GET_LATENCY() != ws) return E_NOT_OK;
    return E_OK;
}
🧠 Quiz Kiểm tra hiểu biết về PLL ›

1. Với HSE = 16 MHz, PLLM = 8, PLLN = 168, PLLP = 2. Tần số SYSCLK là bao nhiêu?

Đúng! f_vco_in = 16/8 = 2 MHz; f_vco = 2×168 = 336 MHz; SYSCLK = 336/2 = 168 MHz. Lưu ý VCO input cần ~1–2 MHz là tốt nhất.

2. Tại sao phải set Flash Latency TRƯỚC khi switch sang PLL?

Đúng! Flash read cycle ~30ns cố định. Ở 168 MHz, CPU cycle chỉ ~6ns. Nếu không có wait state, CPU đọc flash trước khi dữ liệu sẵn sàng → đọc instruction sai → hard fault hoặc undefined behavior.

3. Trong AUTOSAR MCAL, các hằng số PLLM, PLLN, PLLP nên được đặt ở đâu?

Đúng! AUTOSAR yêu cầu tính configurability — mọi tham số cấu hình phải nằm trong file _Cfg.h được sinh ra bởi tool, không được hardcode trong implementation. Điều này cho phép thay đổi cấu hình mà không sửa code.
