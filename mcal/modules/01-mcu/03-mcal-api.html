MCU Driver API (AUTOSAR MCAL)

Tổng hợp toàn bộ AUTOSAR MCU Driver API: các hàm bắt buộc, tùy chọn, kiểu dữ liệu chuẩn, trình tự khởi động đúng, và cách tích hợp vào EcuM (ECU Manager).

📖 Lý thuyết AUTOSAR MCU Driver — Kiến trúc & API ›

1. MCU Driver trong stack AUTOSAR

MCU Driver nằm ở tầng MCAL (Microcontroller Abstraction Layer) — tầng thấp nhất của BSW (Basic Software). Nó trực tiếp điều khiển hardware registers của vi điều khiển.

Application Layer
RTE (Runtime Environment)
BSW — Services Layer (OS, NvM, Com…)
BSW — Hardware Abstraction (ECUAL)
MCAL — MCU Driver ← Bài này
STM32 Hardware Registers

MCU Driver cung cấp dịch vụ:

  • Khởi tạo MCU hardware (clock, power)
  • Phân phối PLL clock tới peripherals
  • Quản lý Reset — đọc lý do reset, thực hiện software reset
  • Quản lý Power Mode — Normal, Sleep, DeepSleep, Halt
  • Cung cấp RAM section initialization

2. Các hàm API bắt buộc (Mandatory)

APIMô tảGọi bởi
Mcu_Init() Khởi tạo MCU driver, áp dụng cấu hình cơ bản EcuM (startup)
Mcu_InitRamSection() Khởi tạo vùng RAM (clear/copy từ flash) EcuM (startup)
Mcu_InitClock() Cấu hình và kích hoạt clock (PLL, prescalers) EcuM (startup)

Mcu_PerformReset() Thực hiện software reset BSW / Application
Mcu_SetMode() Chuyển power mode (Normal → Sleep…) EcuM (run/sleep)

4. Trình tự khởi động đúng (EcuM Startup)

1
Mcu_Init(&McuConfigData)

Áp dụng cấu hình cơ bản: enable GPIO clocks, basic hardware init

2
Mcu_InitRamSection(MCU_RAM_SECTION_0)

Khởi tạo .bss (zero-init) và .data (copy từ flash) — bắt buộc trước khi dùng biến global

3
Mcu_InitClock(MCU_CLOCK_MODE_PLL)

Cấu hình và kích hoạt PLL, set flash latency, switch SYSCLK

4
while(Mcu_GetPllStatus() != MCU_PLL_LOCKED)

Đợi PLL lock (thường ~100μs). KHÔNG skip bước này!

5
Mcu_DistributePllClock()

Phân phối clock PLL tới toàn bộ peripherals — sau bước này mới được dùng SPI, UART…

Lỗi thường gặp: Khởi tạo UART/SPI trước Mcu_DistributePllClock() → peripheral chạy với clock sai (thường là HSI/2 = 8 MHz) → baudrate bị lệch, giao tiếp thất bại.
💻 Code Triển khai đầy đủ MCU Driver API › 
/**
 * Mcu.h — MCU Driver Public API (AUTOSAR 4.x SWS_Mcu)
 */
#ifndef MCU_H
#define MCU_H

#include "Std_Types.h"   /* Std_ReturnType, uint8, uint32 */
#include "Mcu_Cfg.h"

/* ── Kiểu dữ liệu ─────────────────────────────── */
typedef uint8   Mcu_ClockType;
typedef uint8   Mcu_RamSectionType;
typedef uint8   Mcu_ModeType;

typedef enum {
    MCU_PLL_LOCKED    = 0x00U,
    MCU_PLL_UNLOCKED  = 0x01U,
    MCU_PLL_STATUS_UNDEFINED = 0x02U,
} Mcu_PllStatusType;

typedef enum {
    MCU_POWER_ON_RESET    = 0x00U,
    MCU_WATCHDOG_RESET    = 0x01U,
    MCU_SW_RESET          = 0x02U,
    MCU_RESET_UNDEFINED   = 0xFFU,
} Mcu_ResetType;

typedef struct {
    Mcu_ClockType       defaultClockSetting;
    uint8               ramSectionCount;
    const void*         platformConfig;   /* HAL config pointer */
} Mcu_ConfigType;

/* ── API declarations ──────────────────────────── */
void               Mcu_Init                (const Mcu_ConfigType* ConfigPtr);
Std_ReturnType     Mcu_InitRamSection      (Mcu_RamSectionType RamSection);
Std_ReturnType     Mcu_InitClock           (Mcu_ClockType ClockSetting);
Std_ReturnType     Mcu_DistributePllClock  (void);
Mcu_PllStatusType  Mcu_GetPllStatus        (void);
Mcu_ResetType      Mcu_GetResetReason      (void);
void               Mcu_PerformReset        (void);
void               Mcu_SetMode             (Mcu_ModeType McuMode);

#endif /* MCU_H */

/**
 * Mcu.c — MCU Driver Implementation
 * Platform: STM32F407 / STM32 HAL
 */
#include "Mcu.h"
#include "stm32f4xx_hal.h"

static const Mcu_ConfigType* Mcu_ConfigPtr = NULL_PTR;

/* ── Mcu_Init ──────────────────────────────────── */
void Mcu_Init(const Mcu_ConfigType* ConfigPtr)
{
    Mcu_ConfigPtr = ConfigPtr;
    HAL_Init();  /* Reset peripherals, configure SysTick 1ms */

    /* Enable PWR clock — needed for voltage scaling */
    __HAL_RCC_PWR_CLK_ENABLE();
    /* Scale 1: max performance for 168 MHz */
    __HAL_PWR_VOLTAGESCALING_CONFIG(PWR_REGULATOR_VOLTAGE_SCALE1);
}

/* ── Mcu_InitRamSection ────────────────────────── */
Std_ReturnType Mcu_InitRamSection(Mcu_RamSectionType RamSection)
{
    /* On Cortex-M with standard startup, .bss and .data are already
     * initialized by startup.s before main(). This function is kept
     * for AUTOSAR compliance and future multi-core scenarios. */
    (void)RamSection;
    return E_OK;
}

/* ── Mcu_InitClock ──────────────────────────────── */
Std_ReturnType Mcu_InitClock(Mcu_ClockType ClockSetting)
{
    (void)ClockSetting;  /* Single clock config in this implementation */

    RCC_OscInitTypeDef osc = {0};
    RCC_ClkInitTypeDef clk = {0};

    osc.OscillatorType = RCC_OSCILLATORTYPE_HSE;
    osc.HSEState       = RCC_HSE_ON;
    osc.PLL.PLLState   = RCC_PLL_ON;
    osc.PLL.PLLSource  = MCU_PLL_SRC;
    osc.PLL.PLLM       = MCU_PLLM;
    osc.PLL.PLLN       = MCU_PLLN;
    osc.PLL.PLLP       = MCU_PLLP;
    osc.PLL.PLLQ       = MCU_PLLQ;

    if (HAL_RCC_OscConfig(&osc) != HAL_OK) return E_NOT_OK;

    /* Flash latency BEFORE clock switch — see PLL lesson */
    __HAL_FLASH_SET_LATENCY(MCU_FLASH_LATENCY);
    if (__HAL_FLASH_GET_LATENCY() != MCU_FLASH_LATENCY) return E_NOT_OK;

    clk.ClockType      = RCC_CLOCKTYPE_SYSCLK | RCC_CLOCKTYPE_HCLK |
                         RCC_CLOCKTYPE_PCLK1  | RCC_CLOCKTYPE_PCLK2;
    clk.SYSCLKSource   = RCC_SYSCLKSOURCE_PLLCLK;
    clk.AHBCLKDivider  = MCU_AHB_PRESCALER;
    clk.APB1CLKDivider = MCU_APB1_PRESCALER;
    clk.APB2CLKDivider = MCU_APB2_PRESCALER;

    if (HAL_RCC_ClockConfig(&clk, MCU_FLASH_LATENCY) != HAL_OK) return E_NOT_OK;

    return E_OK;
}

/* ── Mcu_GetPllStatus ───────────────────────────── */
Mcu_PllStatusType Mcu_GetPllStatus(void)
{
    if (READ_BIT(RCC->CR, RCC_CR_PLLRDY) != 0U) return MCU_PLL_LOCKED;
    if (READ_BIT(RCC->CR, RCC_CR_PLLON)  == 0U) return MCU_PLL_STATUS_UNDEFINED;
    return MCU_PLL_UNLOCKED;
}

/* ── Mcu_DistributePllClock ─────────────────────── */
Std_ReturnType Mcu_DistributePllClock(void)
{
    if (Mcu_GetPllStatus() != MCU_PLL_LOCKED) return E_NOT_OK;
    /* HAL_RCC_ClockConfig already routed PLL to all buses.
     * Notify application layer if callback is configured. */
    return E_OK;
}

/* ── Mcu_GetResetReason ─────────────────────────── */
Mcu_ResetType Mcu_GetResetReason(void)
{
    uint32 csr = RCC->CSR;
    RCC->CSR |= RCC_CSR_RMVF;  /* Clear reset flags */

    if (csr & RCC_CSR_PORRSTF)   return MCU_POWER_ON_RESET;
    if (csr & RCC_CSR_IWDGRSTF)  return MCU_WATCHDOG_RESET;
    if (csr & RCC_CSR_WWDGRSTF)  return MCU_WATCHDOG_RESET;
    if (csr & RCC_CSR_SFTRSTF)   return MCU_SW_RESET;
    return MCU_RESET_UNDEFINED;
}

/* ── Mcu_PerformReset ───────────────────────────── */
void Mcu_PerformReset(void)
{
    NVIC_SystemReset();  /* ARM Cortex-M system reset */
}

/* ── Mcu_SetMode ────────────────────────────────── */
void Mcu_SetMode(Mcu_ModeType McuMode)
{
    switch (McuMode) {
        case 0x00U:  /* Normal — already running */
            break;
        case 0x01U:  /* Sleep */
            HAL_PWR_EnterSLEEPMode(PWR_MAINREGULATOR_ON, PWR_SLEEPENTRY_WFI);
            break;
        case 0x02U:  /* Stop (deep sleep, retain RAM) */
            HAL_PWR_EnterSTOPMode(PWR_LOWPOWERREGULATOR_ON, PWR_STOPENTRY_WFI);
            break;
        default:
            break;
    }
}

/**
 * EcuM_StartupTwo() — EcuM Startup Phase Two
 * Đây là nơi AUTOSAR EcuM gọi MCU driver API theo đúng thứ tự.
 *
 * Tham chiếu: AUTOSAR_SWS_ECUStateManager, Fig. 7-2 (Startup sequence)
 */
#include "EcuM.h"
#include "Mcu.h"
#include "Port.h"    /* GPIO init */
#include "Os.h"      /* OS StartOS */

/* Config objects (sinh ra bởi tool như ISOLAR-EVE hoặc EB tresos) */
extern const Mcu_ConfigType McuConfigData;

void EcuM_StartupTwo(void)
{
    Mcu_ResetType resetReason;

    /* ── 1. Khởi tạo MCU driver ── */
    Mcu_Init(&McuConfigData);

    /* ── 2. Đọc lý do reset (trước khi clear flags) ── */
    resetReason = Mcu_GetResetReason();
    EcuM_AL_SetProgrammableInterrupts();

    /* ── 3. Khởi tạo RAM sections ── */
    Mcu_InitRamSection(MCU_RAM_SECTION_0);

    /* ── 4. Cấu hình và kích hoạt PLL ── */
    if (Mcu_InitClock(MCU_CLOCK_MODE_PLL) != E_OK) {
        /* Clock init failed — fallback hoặc reset */
        Mcu_PerformReset();
    }

    /* ── 5. Đợi PLL lock (polling, có timeout) ── */
    {
        uint32 timeout = 100000U;
        while ((Mcu_GetPllStatus() != MCU_PLL_LOCKED) && (timeout > 0U)) {
            timeout--;
        }
        if (timeout == 0U) {
            Mcu_PerformReset();  /* PLL không lock — reset */
        }
    }

    /* ── 6. Phân phối clock PLL tới peripherals ── */
    Mcu_DistributePllClock();

    /* ── 7. Khởi tạo GPIO ports (Port Driver) ── */
    Port_Init(&PortConfigData);

    /* ── 8. Khởi tạo các driver BSW khác (SPI, UART…) ── */
    /* Spi_Init(&SpiConfigData); */
    /* Lin_Init(&LinConfigData); */

    /* ── 9. Khởi động OS ── */
    Os_StartOS(OSDEFAULTAPPMODE);

    /* Không return sau StartOS */
}

/**
 * main.c — Entry point AUTOSAR project
 *
 * Lưu ý: Trong kiến trúc AUTOSAR đầy đủ, main() chỉ gọi
 * EcuM_Init() và không chứa application logic.
 * Application logic nằm trong Runnable Entities được
 * schedule bởi OS và RTE.
 */
#include "EcuM.h"

int main(void)
{
    /* EcuM_Init gọi EcuM_StartupOne → EcuM_StartupTwo → StartOS */
    EcuM_Init();

    /* Không bao giờ đến đây trong hệ thống AUTOSAR bình thường */
    for (;;) { }
}

/* ── Nếu KHÔNG dùng EcuM đầy đủ (bare-metal MCAL) ── */
int main_bare_metal(void)
{
    /* Config object */
    static const Mcu_ConfigType cfg = {
        .defaultClockSetting = MCU_CLOCK_MODE_PLL,
        .ramSectionCount     = 1U,
        .platformConfig      = NULL_PTR,
    };

    Mcu_Init(&cfg);
    Mcu_InitRamSection(0U);

    if (Mcu_InitClock(MCU_CLOCK_MODE_PLL) == E_OK) {
        uint32 t = 100000U;
        while (Mcu_GetPllStatus() != MCU_PLL_LOCKED && t--);
        Mcu_DistributePllClock();
    }

    /* System now running at 168 MHz — initialize other drivers */
    /* ... */

    for (;;) {
        /* Main loop / OS scheduler placeholder */
    }
}
🧠 Quiz Kiểm tra MCU Driver API ›

1. Hàm nào phải được gọi TRƯỚC Mcu_DistributePllClock() để đảm bảo clock tới peripherals đúng?

Đúng! PLL phải lock (MCU_PLL_LOCKED) trước khi phân phối clock. Nếu không, peripheral sẽ nhận clock không ổn định hoặc sai tần số. Mcu_DistributePllClock() nên kiểm tra trạng thái này và trả về E_NOT_OK nếu PLL chưa ready.

2. Mcu_GetResetReason() trả về MCU_WATCHDOG_RESET. Điều này có nghĩa là gì trong vận hành?

Đúng! Watchdog reset xảy ra khi hệ thống bị treo (infinite loop, deadlock, hoặc task quá dài) và không kịp refresh watchdog counter trước khi timeout. Đây là cơ chế an toàn quan trọng trong hệ thống nhúng. Log lý do reset để debug.

3. Trong AUTOSAR, main() thông thường chỉ chứa code nào?

Đúng! Trong kiến trúc AUTOSAR, main() chỉ gọi EcuM_Init(), sau đó EcuM quản lý startup sequence và khởi động OS. Application logic chạy trong Runnable Entities được mapping vào OS Tasks bởi RTE — không bao giờ viết logic trong main().
